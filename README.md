# ai-reviewer

> **⚠️ Work in Progress** — This project is under active development. It is not guaranteed to build or run successfully on your machine.

AI-to-AI code review engine. Four LLMs — two style reviewers and two logic reviewers — independently review every diff in parallel via a dual-pair consensus gate. All pairs must agree with high confidence — or the pipeline is blocked.

Designed for **pure-AI coding environments** where code is generated by agents. Acts as a hard CI gate: exit `0` = safe to merge, exit `1` = blocked.

---

## How It Works

```
git diff
  └─► diff::parse_diff
        └─► ast::extract_context (tree-sitter, Rust + Python)
              └─► prompt assembly
                    └─► tokio::join! ──► Style Reviewer A  (default: MiniMax)
                                    ──► Style Reviewer B  (default: DeepSeek)
                                    ──► Logic Reviewer A  (default: MiniMax)
                                    ──► Logic Reviewer B  (default: DeepSeek)
                          └─► consensus::evaluate (dual-pair gate)
                                └─► report::render → review-report.md
```

Each pair must independently reach `Verdict::Pass` with `confidence ≥ threshold`. Any error, timeout, or low-confidence result is treated as `Fail`.

## Supported Providers

Powered by [rig-core](https://github.com/0xPlaygrounds/rig). Any of the four reviewer slots can be assigned to any provider:

| Provider | `--reviewer-N` value | Default model |
|---|---|---|
| MiniMax | `minimax` | `MiniMax-M2.7` |
| DeepSeek | `deepseek` | `deepseek-chat` |
| Anthropic | `anthropic` | `claude-sonnet-4-6` |
| Google Gemini | `gemini` | `gemini-3.1-pro-preview` |
| OpenAI | `openai` | `gpt-5.4` |

## Usage

```bash
# From a file
ai-reviewer --diff changes.diff --policy policy.md --source-root .

# From stdin
git diff HEAD~1 | ai-reviewer --diff - --policy policy.md --source-root .
```

**Core options:**

```
-d, --diff <PATH>           Unified diff file, or "-" for stdin [required]
-p, --policy <PATH>         Security policy Markdown file [required]
-s, --source-root <PATH>    Repo root for AST resolution [default: .]
-t, --threshold <FLOAT>     Confidence gate threshold [default: 0.90]
-o, --output <PATH>         Report output path [default: <source-root>/review-report.md]
    --max-retries <N>        Self-correction retries per model [default: 3]
    --reviewer-timeout <N>   Hard timeout per LLM call in seconds [default: 120] [env: REVIEWER_TIMEOUT]
    --max-diff-lines <N>     Refuse diffs larger than this line count [default: 5000] [env: MAX_DIFF_LINES]
-v, --verbose                Enable debug tracing
```

**Per-reviewer options** (N = 1–4; 1 & 2 = style pair, 3 & 4 = logic pair):

```
--reviewer-N <PROVIDER>      Provider: minimax, deepseek, anthropic, gemini, openai
                             [env: REVIEWER_N]
--reviewer-N-model <ID>      Model ID override [env: REVIEWER_N_MODEL]
--reviewer-N-api-key <KEY>   API key [env: REVIEWER_N_API_KEY]
--reviewer-N-base-url <URL>  Base URL override for OpenAI-compat providers [env: REVIEWER_N_BASE_URL]
```

**Legacy env vars** (still supported as fallback for reviewers 1/3 and 2/4):

```
MINIMAX_API_KEY / MINIMAX_BASE_URL
DEEPSEEK_API_KEY / DEEPSEEK_BASE_URL
```

## Observability

### Prometheus Metrics

Six metrics are recorded per run and exported to one or both sinks at the end. Both sinks are optional and independently configured.

| Metric | Type | Labels | Description |
|---|---|---|---|
| `ai_reviewer_review_duration_seconds` | Histogram | `reviewer`, `focus` | End-to-end latency per reviewer slot, including retries and backoff |
| `ai_reviewer_attempts_total` | Counter | `reviewer`, `focus`, `outcome` | LLM completion attempts. `outcome` is `success`, `timeout`, or `max_retries` |
| `ai_reviewer_confidence` | Histogram | `reviewer`, `focus`, `verdict` | Model confidence score distribution on successful parses |
| `ai_reviewer_findings_total` | Counter | `severity`, `reviewer`, `focus` | Total code findings raised |
| `ai_reviewer_diff_lines` | Gauge | — | Lines in the reviewed diff |
| `ai_reviewer_gate_passed` | Gauge | — | `1` if the consensus gate passed, `0` if it failed |

**Sink A — textfile collector** (for Prometheus node_exporter):

```bash
export METRICS_FILE_PATH=/var/lib/node_exporter/textfile/ai_reviewer.prom
```

**Sink B — Pushgateway**:

```bash
export PROMETHEUS_PUSHGATEWAY_URL=http://pushgateway:9091
# Metrics are pushed to: <URL>/metrics/job/ai-reviewer
```

### Distributed Tracing (OpenTelemetry)

When `OTEL_EXPORTER_OTLP_ENDPOINT` is set, every run exports trace spans via OTLP/HTTP to your collector (Jaeger, Grafana Tempo, Datadog, etc.). Each reviewer call becomes a child span with structured attributes, making it possible to see exactly where time was spent and what each model returned.

```
run ──────────────────────────────────────────── 187s
  ├─ reviewer.call  reviewer=MiniMax  focus=style ── 45s  verdict=pass  confidence=0.94  attempts=1
  ├─ reviewer.call  reviewer=DeepSeek focus=style ── 52s  verdict=pass  confidence=0.91  attempts=1
  ├─ reviewer.call  reviewer=MiniMax  focus=logic ── 38s  verdict=fail  confidence=0.87  attempts=2
  └─ reviewer.call  reviewer=DeepSeek focus=logic ── 51s  verdict=pass  confidence=0.93  attempts=1
```

**Setup:**

```bash
export OTEL_EXPORTER_OTLP_ENDPOINT=http://your-otel-collector:4318
export OTEL_SERVICE_NAME=ai-reviewer   # optional, this is the default
```

If the endpoint is unreachable or the env var is not set, the tool falls back to normal log-only output — tracing is non-critical and never blocks a review run.

**Span attributes recorded on each `reviewer.call` span:**

| Attribute | Example value |
|---|---|
| `reviewer` | `MiniMax` |
| `focus` | `Style` |
| `verdict` | `pass` |
| `confidence` | `0.94` |
| `attempts` | `1` |

## The Policy File

A Markdown file injected into the LLM system prompt as your ruleset. Example:

```markdown
## Security Rules
- SEC-001: Never log credentials, tokens, or PII
- SEC-002: All SQL queries must use parameterized statements
- SEC-003: File paths must be validated before use

## Code Quality Rules
- LOGIC-001: Every error must be handled or explicitly propagated
- LOGIC-002: Async tasks must not hold locks across await points
```

Models must reference rule IDs in their findings (e.g. `"rule_id": "SEC-001"`).

---

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | All pairs passed with confidence ≥ threshold |
| `1` | Gate blocked — conflict, low confidence, defects found, or timeout |
| `2` | Fatal error — bad paths, missing API keys, diff too large |

A Markdown report is always written to `--output` (default: `<source-root>/review-report.md`).

## For AI Agents

```bash
# 1. Generate diff
git diff HEAD > /tmp/changes.diff

# 2. Run review
ai-reviewer --diff /tmp/changes.diff --policy policy.md --source-root . --output /tmp/report.md

# 3. Act on exit code
# exit 0 → proceed
# exit 1 → read /tmp/report.md, fix findings at indicated locations, retry
# exit 2 → fix configuration (missing keys, diff too large), do not retry blindly
```

Findings in the report are structured as: `Severity | Rule ID | File:Line | Description | Suggestion`.
